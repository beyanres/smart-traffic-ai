## 📑 Abstract
This project presents a web-based simulation of an intelligent traffic intersection, designed to demonstrate the critical concepts of **Deadlock Prevention** in Operating Systems. By mapping real-world traffic flows to OS resource management, the simulation provides a visual representation of system synchronization, state transitions, and the prevention of the "Circular Wait" condition.

## 🧠 Theoretical Foundations
To prevent Deadlock in computer systems, we must break one of the four **Coffman Conditions**. This simulation focuses on:

| Concept | Simulation Logic | OS Theory |
| :--- | :--- | :--- |
| **Deadlock** | Traffic Gridlock (Cars blocked in all directions) | Circular Wait / Hold & Wait |
| **Resource** | Intersection Road Segment | Critical Section |
| **Prevention** | Traffic Signal Sequencing | Resource Ordering / Preemption |

*   **Gridlock (Deadlock):** Occurs when all cars enter the intersection simultaneously, satisfying the *Circular Wait* condition, leading to system failure.
*   **Prevention Strategy:** The simulation enforces a strict scheduling algorithm (Resource Ordering) to ensure that only authorized directions proceed, effectively breaking the deadlock cycle before it occurs.

## 🏗 System Architecture
The project is built as a lightweight, interactive state-machine simulation:

### 1. The Logic Engine (JavaScript)
- **State Management:** Handles the lifecycle of traffic signals (Green/Red transitions).
- **Collision Detection (Prevention Mode):** Prevents the scheduling of conflicting threads (cars) when the system resources (intersection) are busy.
- **Asynchronous Execution:** Utilizes `setTimeout` and `CSS animations` to simulate parallel processes entering and exiting critical sections.

### 2. Frontend & UI (CSS/HTML)
- **Responsive Layout:** Optimized for browser-based presentation.
- **CSS Animations:** Implements smooth transitions for "process" (car) movement, simulating execution flow.

## 🚀 Usage Guide
1.  **Open the Application:** Navigate to the [Live Simulation Link](https://beyanres.github.io/smart-traffic-ai/).
2.  **Simulation Modes:**
    - **Prevention Enabled:** The logic enforces strict signaling protocols. Observe how cars pass safely.
    - **Prevention Disabled:** The system allows all threads to enter the critical section simultaneously. Observe the Gridlock (Deadlock) outcome.
3.  **Observation:** Use the browser console or the UI logs to analyze the state transitions of the traffic signals.
